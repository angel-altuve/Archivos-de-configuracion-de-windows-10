<h1><a id="user-content-resumen" class="anchor" aria-hidden="true" href="#resumen"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Resumen</h1>
<p>Esta guía es una recopilación de todos los pasos necesarios para construir un
entorno de escritorio a partir de una instalación limpia basada en Arch Linux.
Voy a asumir que te manejas bien con sistemas operativos basados en Linux y sus
líneas de comandos. Ya que estás leyendo esto, asumiré también que has visto
algunos vídeos de "<em>tiling window managers</em>" en Youtube, porque ahí es donde
empieza el sinfín. Puedes elegir el gestor de ventanas que quieras, pero aquí
usaremos Qtile como primer "<em>tiling window manager</em>", dado que fue con el que
empecé yo. Esta guía es básicamente una descripción de cómo he construido mi
entorno de escritorio desde 0.</p>
<h1><a id="user-content-instalación-de-arch-linux" class="anchor" aria-hidden="true" href="#instalación-de-arch-linux"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Instalación de Arch Linux</h1>
<p>El punto de partida de esta guía es justo después de una instalación basada en
Arch completa y limpia. La
<strong><a href="https://wiki.archlinux.org/index.php/Installation_guide" rel="nofollow">Wiki de Arch</a></strong>
no te dice qué hacer después de establecer la contraseña del superusuario,
sugiere instalar un cargador de arranque, pero antes de eso yo me aseguraría de
tener internet:</p>
<div class="highlight highlight-source-shell"><pre>pacman -S networkmanager
systemctl <span class="pl-c1">enable</span> NetworkManager</pre></div>
<p>Ahora puedes instalar un cargador de arranque y probarlo de forma "segura", así
es como se haría en hardware moderno,
<a href="https://wiki.archlinux.org/index.php/Installation_guide#Example_layouts" rel="nofollow">suponiendo que has montado la partición efi en /boot</a>:</p>
<div class="highlight highlight-source-shell"><pre>pacman -S grub efibootmgr os-prober
grub-install --target=x86_64-efi --efi-directory=/boot
os-prober
grub-mkconfig -o /boot/grub/grub.cfg</pre></div>
<p>Ahora puedes crear tu usuario:</p>
<div class="highlight highlight-source-shell"><pre>useradd -m username
passwd username
usermod -aG wheel,video,audio,storage username</pre></div>
<p>Para tener privilegios de superusuario necesitamos sudo:</p>
<div class="highlight highlight-source-shell"><pre>pacman -S sudo</pre></div>
<p>Edita <strong>/etc/sudoers</strong> con <em>nano</em> o <em>vim</em> y descomenta la línea con "wheel":</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span># Uncomment to allow members of group wheel to execute any command</span>
<span class="pl-c"><span class="pl-c">#</span> %wheel ALL=(ALL) ALL</span></pre></div>
<p>Ahora ya puedes reiniciar:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Sal de la imagen ISO, desmóntala y extráela</span>
<span class="pl-c1">exit</span>
umount -R /mnt
reboot</pre></div>
<p>Después de haber iniciado sesión, el internet debería funcionarte sin problema,
pero eso solo aplica si tu ordenador está conectado por cable. Si estás en un
portátil que no tiene puertos Ethernet, probablemente hayas usado
<strong><a href="https://wiki.archlinux.org/index.php/Iwd#iwctl" rel="nofollow">iwctl</a></strong>
durante la instalación, pero este programa ya no está disponible a no ser que
lo hayas instalado explícitamente. Sin embargo, tenemos
<strong><a href="https://wiki.archlinux.org/index.php/NetworkManager" rel="nofollow">NetworkManager</a></strong>,
así que no hay problema, para conectarte a una red inalámbrica con este software
solo debes hacer esto:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Lista las redes disponibles</span>
nmcli device wifi list
<span class="pl-c"><span class="pl-c">#</span> Conéctate a tu red</span>
nmcli device wifi connect TU_SSID password TU_CONTRASEÑA</pre></div>
<p>Échale un vistazo a
<a href="https://wiki.archlinux.org/index.php/NetworkManager#nmcli_examples" rel="nofollow">esta página</a>
para otras opciones proporcionadas por <em>nmcli</em>. Lo último que tenemos que hacer
antes de pensar en entornos de escritorio es instalar
<strong><a href="https://wiki.archlinux.org/index.php/Xorg" rel="nofollow">Xorg</a></strong>:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S xorg</pre></div>
<h1><a id="user-content-inicio-de-sesión-y-gestor-de-ventanas" class="anchor" aria-hidden="true" href="#inicio-de-sesión-y-gestor-de-ventanas"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Inicio de sesión y gestor de ventanas</h1>
<p>Primero, necesitamos una forma de iniciar sesión y abrir programas como
navegadores y terminales, así que empezaremos instalando
<strong><a href="https://wiki.archlinux.org/index.php/LightDM" rel="nofollow">lighdm</a></strong>
y <strong><a href="https://wiki.archlinux.org/index.php/Qtile" rel="nofollow">qtile</a></strong>.
<em>lightdm</em> no funcionará si no instalamos también un
<strong><a href="https://wiki.archlinux.org/index.php/LightDM#Greeter" rel="nofollow">greeter</a></strong>.
También necesitamos
<strong><a href="https://wiki.archlinux.org/index.php/Xterm" rel="nofollow">xterm</a></strong>
porque esa es la terminal que <em>qtile</em> abre por defecto, hasta que lo cambiemos
en el archivo de configuración. Para editar archivos de configuración
necesitaremos también un editor de texto, puedes usar
<strong><a href="https://wiki.archlinux.org/index.php/Visual_Studio_Code" rel="nofollow">vscode</a></strong>
o directamente
<strong><a href="https://wiki.archlinux.org/index.php/Neovim" rel="nofollow">neovim</a></strong>
si tienes experiencia previa, si no no lo recomiendo. Por último necesitamos un
navegador.</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S lightdm lightdm-gtk-greeter qtile xterm code firefox</pre></div>
<p>Activa el servicio de <em>lightdm</em> y reinicia el ordenador, deberías poder iniciar
sesión en Qtile a través de <em>lightdm</em>.</p>
<div class="highlight highlight-source-shell"><pre>sudo systemctl <span class="pl-c1">enable</span> lightdm
reboot</pre></div>
<h1><a id="user-content-configuración-básica-de-qtile" class="anchor" aria-hidden="true" href="#configuración-básica-de-qtile"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Configuración básica de Qtile</h1>

<p>Antes de hacer nada, si no tienes la distribución del teclado en inglés,
deberías cambiarla usando <em>setxkbmap</em>. Abre <em>xterm</em> con <strong>mod + enter</strong>, y
cambia la distribución a español:</p>
<div class="highlight highlight-source-shell"><pre>setxkbmap es</pre></div>
<p>Ten en cuenta que este cambio no es permanente, si reinicias el PC tendrás que
esribir el comando anterior de nuevo. Consulta <a href="#xprofile">esta sección</a> para
hacer cambios permanentes o sigue el orden natural de esta guía si tienes
tiempo suficiente.</p>
<p>Por defecto, no hay menú, tienes que lanzar programas a través de <em>xterm</em>. En
este punto puedes instalar otro emulador de terminal si lo prefieres:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Instala otro de tu preferencia</span>
sudo pacman -S alacritty</pre></div>
<p>Abre el archivo de configuración de Qtile:</p>
<div class="highlight highlight-source-shell"><pre>code <span class="pl-k">~</span>/.config/qtile/config.py</pre></div>
<p>Al principio, después de los imports, encontrarás una lista llamada <em>keys</em>, que
contiene la línea siguiente:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-v">Key</span>([<span class="pl-s1">mod</span>], <span class="pl-s">"Return"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(<span class="pl-s">"xterm"</span>)),</pre></div>
<p>Edítala para lanzar el emulador de terminal que has instalado:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-v">Key</span>([<span class="pl-s1">mod</span>], <span class="pl-s">"Return"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(<span class="pl-s">"alacritty"</span>)),</pre></div>
<p>Instala un menú como
<strong><a href="https://wiki.archlinux.org/index.php/Dmenu" rel="nofollow">dmenu</a></strong>
o <strong><a href="https://wiki.archlinux.org/index.php/Rofi" rel="nofollow">rofi</a></strong>:</p>

<p>Después añade atajos de teclado para el menú:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-v">Key</span>([<span class="pl-s1">mod</span>], <span class="pl-s">"m"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(<span class="pl-s">"rofi -show run"</span>)),

<div class="highlight highlight-source-shell"><pre>sudo pacman -S which
rofi-theme-selector</pre></div>

<p>En este punto, no hay audio, necesitamos
<strong><a href="https://wiki.archlinux.org/index.php/PulseAudio" rel="nofollow">pulseaudio</a></strong>.
Recomiendo instalar un programa gráfico para manejar el audio como

<p>En Arch,
<a href="https://wiki.archlinux.org/index.php/PulseAudio#Running" rel="nofollow">pulseaudio está activado por defecto</a>,
pero puede que tengas que reiniciar para que <em>pulseaudio</em> arranque. Después de
reiniciar, abre <em>pavucontrol</em> usando <em>rofi</em>, activa el audio (porque está en

<p>Ahora puedes establecer atajos de teclado para <em>pulseaudio</em>, abre el archivo de

<span class="pl-v">Key</span>([], <span class="pl-s">"XF86AudioRaiseVolume"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(

)),
<span class="pl-v">Key</span>([], <span class="pl-s">"XF86AudioMute"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(
    <span class="pl-s">"pactl set-sink-mute @DEFAULT_SINK@ toggle"</span>
)),</pre></div>
<p>Aunque para una mejor experiencia en la línea de comandos, recomiendo usar
<strong><a href="https://www.archlinux.org/packages/community/x86_64/pamixer/" rel="nofollow">pamixer</a></strong>:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S pamixer</pre></div>

<div class="highlight highlight-source-python"><pre><span class="pl-c"># Volumen</span>

<span class="pl-v">Key</span>([], <span class="pl-s">"XF86AudioRaiseVolume"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(<span class="pl-s">"pamixer --increase 5"</span>)),
<span class="pl-v">Key</span>([], <span class="pl-s">"XF86AudioMute"</span>, <span class="pl-s1">lazy</span>.<span class="pl-en">spawn</span>(<span class="pl-s">"pamixer --toggle-mute"</span>)),</pre></div>

<h2><a id="user-content-monitores" class="anchor" aria-hidden="true" href="#monitores"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Monitores</h2>
<p>Si tienes múltiples monitores, seguramente quieras usarlos todos. Así es como
funciona <strong><a href="https://wiki.archlinux.org/index.php/Xrandr" rel="nofollow">xrandr</a></strong>:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Lista todas las salidas y resoluciones disponibles</span>
xrandr
<span class="pl-c"><span class="pl-c">#</span> Formato común para un portátil con monitor extra</span>
xrandr --output eDP-1 --primary --mode 1920x1080 --pos 0x1080 --output HDMI-1 --mode 1920x1080 --pos 0x0</pre></div>
<p>Es necesario especificar la posición de cada salida, si no se utilizará 0x0, y
todas las salidas estarán solapadas. Ahora bien, si no quieres calcular píxeles
y demás necesitas una interfaz gráfica como
<strong><a href="https://www.archlinux.org/packages/community/any/arandr/" rel="nofollow">arandr</a></strong>:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S arandr</pre></div>
<p>Ábrela con <em>rofi</em>, ordena las pantallas como quieras, y después puedes guardar
la disposición de las mismas, lo cual simplemente te dará un script con el
comando exacto de <em>xrandr</em> que necesitas. Guarda ese script, pero todavía no
le des al botón de aplicar.</p>
<p>Para un sistema con múltiples monitores deberías crear una instancia de <em>Screen</em>
por cada uno de ellos en la configuración de Qtile.</p>
<p>Encontrarás una lista llamada <em>screens</em> en la configuración de Qtile que
contiene solo un objeto inicializado con una barra en la parte de abajo.
Dentro de esa barra puedes ver los widgets con los que viene por defecto.</p>
<p>Añade tantas pantallas como necesites y copia-pega los widgets, más adelante
podrás personalizarlos. Ahora puedes volver a <em>arandr</em>, darle click en "apply"
y reiniciar el gestor de ventanats.</p>
<p>Con esto tus monitores deberían funcionar.</p>

<p>Como he mencionado antes, estos cambios no son permanentes. Para que lo sean
necesitamos un par de cosas. Primero instala
<strong><a href="https://wiki.archlinux.org/index.php/Xinit" rel="nofollow">xinit</a></strong>:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S xorg-xinit</pre></div>
<p>Ahora puedes usar <em>~/.xprofile</em> para lanzar programas antes de que se ejecute
el gestor de ventanas:</p>
<div class="highlight highlight-source-shell"><pre>touch <span class="pl-k">~</span>/.xprofile</pre></div>
<p>Por ejemplo, si escribes esto en tu <em>~/.xprofile</em>:</p>
<div class="highlight highlight-source-shell"><pre>xrandr --output eDP-1 --primary --mode 1920x1080 --pos 0x1080 --output HDMI-1 --mode 1920x1080 --pos 0x0 <span class="pl-k">&amp;</span>
setxkbmap es <span class="pl-k">&amp;</span>

volumeicon <span class="pl-k">&amp;</span>

<p>Si no quieres usar <em>rm</em> constantemente y arriesgarte a perder ficheros,
necesitas un sistema de basura. Por suerte, es bastante sencillio de hacer
<a href="https://wiki.archlinux.org/index.php/Trash_management#Trash_creation" rel="nofollow">usando alguna de estas herramientas</a>

<div class="highlight highlight-source-shell"><pre>sudo pacman -S glib2 gvfs

<p>El momento que has estado esperando ha llegado, finalmente vas a instalar un
tema oscuro. Yo uso <em>Material Black Colors</em>, puedes descargar una versión
<a href="https://www.gnome-look.org/p/1316887/" rel="nofollow">aquí</a>, con sus respectivos iconos
<a href="https://www.pling.com/p/1333360/" rel="nofollow">aquí</a>.</p>
<p>Sugiero empezar con
<em>Material-Black-Blueberry</em> y <em>Material-Black-Blueberry-Suru</em>. Puedes encontrar
otros temas para GTK
<a href="https://www.gnome-look.org/browse/cat/135/" rel="nofollow">en esta página</a>.
Una vez tengas descargados los temas, puedes hacer esto:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Asumiendo que has descargado Material-Black-Blueberry</span>
<span class="pl-c1">cd</span> Downloads/
sudo pacman -S unzip
unzip Material-Black-Blueberry.zip
unzip Material-Black-Blueberry-Suru.zip
rm Material-Black<span class="pl-k">*</span>.zip

<span class="pl-c"><span class="pl-c">#</span> Haz tu tema visible a GTK</span>
sudo mv Material-Black-Blueberry /usr/share/themes
sudo mv Material-Black-Blueberry-Suru /usr/share/icons</pre></div>
<p>Ahora edita <strong>~/.gtkrc-2.0</strong> y <strong>~/.config/gtk-3.0/settings.ini</strong> añdiendo
estas líneas:</p>
<div class="highlight highlight-source-ini"><pre><span class="pl-c"><span class="pl-c">#</span> ~/.gtkrc-2.0</span>
<span class="pl-k">gtk-theme-name</span> = <span class="pl-s"><span class="pl-pds">"</span>Material-Black-Blueberry<span class="pl-pds">"</span></span>
<span class="pl-k">gtk-icon-theme-name</span> = <span class="pl-s"><span class="pl-pds">"</span>Material-Black-Blueberry-Suru<span class="pl-pds">"</span></span>

<span class="pl-c"><span class="pl-c">#</span> ~/.config/gtk-3.0/settings.ini</span>
<span class="pl-k">gtk-theme-name</span> = Material-Black-Blueberry
<span class="pl-k">gtk-icon-theme-name</span> = Material-Black-Blueberry-Suru</pre></div>
<p>La próxima vez que inicies sesión verás los cambios aplicados. Puedes instalar
también un tema de cursor distinto, para ello necesitas
<strong><a href="https://www.archlinux.org/packages/extra/x86_64/xcb-util-cursor/" rel="nofollow">xcb-util-cursor</a></strong>.
El tema que yo uso es
<a href="https://www.gnome-look.org/p/999927/" rel="nofollow">Breeze</a>, descárgalo, y después:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S xcb-util-cursor
<span class="pl-c1">cd</span> Downloads/
tar -xf Breeze.tar.gz
sudo mv Breeze /usr/share/icons</pre></div>
<p>Edita <strong>/usr/share/icons/default/index.theme</strong> añadiendo esto:</p>
<div class="highlight highlight-source-ini"><pre><span class="pl-en">[Icon Theme]</span>
<span class="pl-k">Inherits</span> = Breeze</pre></div>
<p>Ahora vuelve a editar <strong>~/.gtkrc-2.0</strong> y <strong>~/.config/gtk-3.0/settings.ini</strong>:</p>
<div class="highlight highlight-source-ini"><pre><span class="pl-c"><span class="pl-c">#</span> ~/.gtkrc-2.0</span>
<span class="pl-k">gtk-cursor-theme-name</span> = <span class="pl-s"><span class="pl-pds">"</span>Breeze<span class="pl-pds">"</span></span>

<span class="pl-c"><span class="pl-c">#</span> ~/.config/gtk-3.0/settings.ini</span>
<span class="pl-k">gtk-cursor-theme-name</span> = Breeze</pre></div>
<p>Asegurate de escribir bien los nombres de los temas e iconos, deben ser
exactamente los nombres de los directorios donde se encuentran, los que
ofrece esta salida:</p>
<div class="highlight highlight-source-shell"><pre>ls /usr/share/themes
ls /usr/share/icons</pre></div>
<p>Recuerda que solo verás los cambios si inicias sesión de nuevo. También hay
herramientas gráficas para cambiar temas, yo simplemente prefiero la forma
tradicional de editar ficheros, pero puedes usar
<strong><a href="https://www.archlinux.org/packages/community/x86_64/lxappearance/" rel="nofollow">lxappearance</a></strong>,
que es un programa independiente del entorno de escritorio para realizar esta
tarea, y te permie previsualizar los temas.</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S lxappearance</pre></div>
<p>Finalmente, si quieres transparencia y demás instala un compositor:</p>
<div class="highlight highlight-source-shell"><pre>sudo pacman -S picom
<span class="pl-c"><span class="pl-c">#</span> Pon esto en ~/.xprofile</span>
picom <span class="pl-k">&amp;</span></pre></div>
